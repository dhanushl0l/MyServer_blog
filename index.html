My-Server: Universal Device Sync Platform

Sync your devices, whether local or in the cloud.

Overview

My-Server is a lightweight, self-hosted server that enables seamless data synchronization across all your devices—be it Windows, Linux, or Android. Whether you are looking to sync files, manage notifications, clipboard data, or even SMS, My-Server handles it all.

Inspired by the simplicity of Docker but designed for users of all technical levels, My-Server allows you to run your own server on anything from a local network to the cloud, ensuring total privacy and control over your data.

Why Device Sync Matters

Every day, we use multiple devices—phones, tablets, laptops—and keeping them in sync can be frustrating. My-Server solves this challenge by providing a universal sync platform:

  • No more missed notifications: Manage calls, SMS, and notifications across all devices.
  • Unified clipboard: Copy text on one device, paste it on another.
  • Seamless file sync: Update documents across all your platforms instantly.
  • Password & search sync: Keep your data available on all devices.

Key Features

  • File Sync: Keep your files updated across all devices.
  • Notification Sync: Sync notifications across devices.
  • Clipboard Sync: Instant clipboard sync between devices.
  • Phone & SMS Management: Manage calls and messages from any device.
  • Password and Search Sync: Access your saved data on all platforms.

Architecture

Client-side Encryption: All data sent to the server is encrypted client-side, ensuring privacy. Decryption happens locally, giving you full control over your sensitive information.

Server Flexibility: While the current implementation uses a Flask server, this could be replaced by another backend in the future. Our primary focus is flexibility and user control.

Rust Apps: The core apps are written in Rust, known for its performance and safety. Each app is designed to run efficiently across all platforms.

Getting Started

Requirements:

  • Python/Flask: The server backend (optional if using another backend in the future).
  • Hardware: You can host a Flask app on a PC, an old laptop, an EC2 server, or any cloud server, such as PythonAnywhere.

Security

All data is encrypted on the client-side before transmission, ensuring that the server only handles storing data. Decryption is handled by client to maintain security.
